介绍

本笔记针对Policy Gradient算法展开，所以在讲Baseline的问题之前需要先回顾一下Policy Gradient的内容：

  • 首先有策略网络Policy Network
  • 然后有状态价值函数State-Value Function

  • 最后有策略梯度Policy Gradient

实际上式的Policy Gradient方差较大，不利于训练，而Baseline的目的就是降低Policy Gradient的方差，加快收敛。

这里会有一个疑问，既然取任何值都不影响Policy Gradient，那么是不是根本没有必要设计Baseline。答案是否，并且Baseline的引入非常有必要。因为在实际中，我们并没有真的计算期望本身（期望中有积分号，通常无法求解），而是使用了蒙特卡洛方法近似，虽然不影响期望，但却能影响蒙特卡洛近似后的Policy Gradient。如果选择的比较好，则会使近似值比较接近，进而让蒙特卡洛近似的方差降低，算法收敛速度加快。
