Math Pre-Training

数据收集和清洗

要训练数学模型，那么必然需要一个数学相关的数据集，本文中作者在Common Crawl数据集的基础上建立了一个专门的数据集，称之为DeepSeekMath Corpus。

We present an iterative pipeline that demonstrates how to systematically gather a large-scale mathematical corpus from Common Crawl, starting with a seed corpus (e.g., a small but high-quality collection of math-related dataset).

为了构造高质量的数学数据集，作者建立了一套 迭代式的数据收集和清洗过程 ，将一个规模较小的高质量数学相关的数据集作为种子数据集，从Common Crawl数据集中系统性的收集到大规模数学数据集。整个数据收集和清洗过程通过fastText模型完成，整个过程分为四轮进行。

第一轮：

  1. 选择一个高质量的数学类数据集作为整个Pipeline的初始化种子数据集，作者选择了OpenWebMath数据集
  2. 从OpenWebMath数据集中选择500000个样本作为正样本，从Common Crawl数据集中选择500000个样本作为负样本
  3. 使用第2步产生的正负样本数据训练fastText模型，用来识别类似OpenWebMath的网页数据，下面是训练相关的参数设置
    1. Training Platform: open-source library, https://fasttext.cc
    2. Vector dimension: 256
    3. Learning rate: 0.1
    4. The maximum length of word n-gram: 3
    5. The minimum number of word occurrences: 3
    6. The number of training epochs: 3
  4. 使用URL-based deduplication和Near-deduplication技术对Common Crawl数据集做去重，最终生成40B的HTML网页
  5. 使用fastText模型从去重后的Common Crawl数据集中收集数学相关的网页
    1. 使用fastText模型对Common Crawl中的数据评分
    2. 按照评分从高到底排序
    3. 按照排名划分前40B、80B、120B、160B个Token数量的位置
    4. 选择上面4种情况的Token数据进行预训练，通过性能决定使用哪个量级的数据集

第二/三/四轮：

  1. 根据域名将Common Crawl数据集切分为不相交的集合
  2. 计算每个集合中已被前一轮收集的网页的占比，占比超过10%的集合标记为数学相关的集合
  3. 从数学相关的集合中将未被收集的网页数据加入到种子数据集
  4. 继续强化fastText模型，训练过程与第一轮类似
  5. 使用fastText模型从去重后的Common Crawl数据集中收集数学相关的网页，筛选过程与第一轮类似

After four iterations of data collection, we end up with 35.5M mathematical web pages, totaling 120B tokens. In the fourth iteration, we notice that nearly 98% of the data has already been collected in the third iteration, so we decide to cease data collection.

作者表示，在第四轮进行中可以知道经过三轮过滤之后，已经有98%的数学相关的数据已被收集，因此经过第四轮清洗之后停止数据收集与清洗的过程。最终生成35.5M个数学相关网页，Token总数达到120B。

SFT

SFT Data

We construct a mathematical instruction-tuning dataset covering English and Chinese problems from different mathematical fields and of varying complexity levels: problems are paired with solutions in chain-of-thought (CoT) (Wei et al., 2022), program-of-thought (PoT) (Chen et al., 2022; Gao et al., 2023), and tool-integrated reasoning format (Gou et al., 2023).

作者建立了一个数学指令微调数据集(Mathematical Instruction-tuning Dataset)，包含了不同数学领域的、各种各样形式的中英文数据，数据集的样本数(问题数)是776K。这里的各种各样形式首先是问题答案对，其次是答案格式包含Chain-of-thought (CoT)、Program-of-thought (PoT)、Tool-integrated Reasoning Format这些格式。

数据集中的数据主要分为以下几部分：

  • 英文数据集
    • 将GSM8K和MATH数据集标注为Tool-integrated的形式
    • 加上MathInstruct的子集和Lila-00D的训练集，这里面包含了CoT和PoT格式的数据
    • 数据涵盖了各式各样的数学领域，比如线性代数、概率论、数论、微积分、几何等等
  • 中文数据集
    • 将Chinese K-12数学问题标注成CoT和Tool-integrated的形式
    • Chinese K-12数学问题分为76个子主题，比如线性方程等等

DeepSeekMath-Instruct 7B 的训练和评估

DeepSeekMath-Instruct-7B在DeepSeekMath-Base基础上进行微调，训练过程中使用Sequence Packing[2]方法，将样本随机的拼接到一起，直到到达4K Context Length的上限，模型训练500个Step，一个Batch大小是256，学习率设定为常数5e-5。

作者在4个Benchmark (GSM8K, MATH, MGSM-zh, CMATH)上进行评估，评估分为两部分：不使用工具和使用工具。

对于不使用工具的评估，性能全面超越开源模型，赢过部分闭源模型；对于使用工具的评估，同样全面超越开源模型。

As the value function employed in PPO is typically another model of comparable size as the policy model, it brings a substantial memory and computational burden. Additionally, during RL training, the value function is treated as a baseline in the calculation of the advantage for variance reduction. While in the LLM context, usually only the last token is assigned a reward score by the reward model, which may complicate the training of a value function that is accurate at each token.

作者在提出GRPO之前先给出了PPO算法的缺陷：

  1. Value Network的大小与Policy Network的大小相当，所以给训练带来了大量的Memory和计算成本
  2. 计算代理目标函数时需要用到每个Token的Advantage，训练一个能够准确估计每个Token的Reward Model和Value Model的复杂度很高

To address this, as shown in Figure 4, we propose Group Relative Policy Optimization (GRPO), which obviates the need for additional value function approximation as in PPO, and instead uses the average reward of multiple sampled outputs, produced in response to the same question, as the baseline.

为了解决上面的缺陷，作者提出了GRPO算法，去除了Value Network。

结果监督

如果使用结果监督的方式进行GRPO的训练，则按照如下的过程进行

  1. 从数据集中采样一个问题
  2. 从旧策略中采样到个答案
  3. 使用奖励模型对答案进行评分，生成个奖励
  4. 计算Advantage
  5. 使用这些数据最大化GRPO目标函数

可以看到，结果监督的训练方式中，奖励模型只会对完整的输出进行评分并输出一个奖励，因此1个答案中所有Token的Advantage都是相同的。

过程监督

采用结果监督的方式训练不适用于复杂的数学任务，因为它无法做到对推理过程的有效监督，所以作者还探索了过程监督的训练方式，过程如下：

  1. 从数据集中采样一个问题
  2. 从旧策略中采样到个答案
  3. 使用过程奖励模型对输出的每个Token进行评分，输出每个Token对应的奖励，于是最终产生组奖励
  4. 对每个奖励进行归一化
  5. 计算Advantage
  6. 使用这些数据最大化GRPO目标函数

关于奖励估计

本文中不管是结果监督还是过程监督，Advantage都是奖励本身，而Advantage的原始定义是 ，其中状态价值 是Baseline，起到了缓解 值过估计的目的。因为本文去除了Value Network，所以无法得到状态价值 ，只能通过奖励得到 值估计，而原始定义的Advantage则无法计算。因此这里作者选择对奖励进行均值方差归一化，以达到Baseline同样的效果。

迭代式RL

作者指出，随着RL训练的进行，旧的奖励模型对新的Policy的监督也许不再有效，因此作者提出了Iterative RL with GRPO的方法。这个方法的思想很简单，在每个Iteration结束之前，基于当前Iteration采样到的数据生成奖励模型的训练集，然后继续训练奖励模型。在下一个Iteration中，使用新的奖励模型对Policy的输出进行评分。奖励模型的训练采用Replay Buffer机制，Replay保留10%的历史数据。

作者比较了Online RFT和RFT的训练性能，实验结果表明，Online RFT性能优于RFT。从实验结果的细节可以看到，在训练初期Online RFT和RFT性能相当，但随着训练的继续，Online RFT的性能逐步超越RFT性能。作者认为，Online RFT训练开始时Policy Model使用SFT Model进行初始化，因此在训练初期，Online RFT采样到的数据与从SFT Model采样的数据差异不大，所以两者性能相当。但当训练中后期，从Policy Model采样到的数据优于从SFT Model采样到的数据，因此Online RFT的性能会超越RFT的性能。

Gradient Coefficient

不同的训练方法中使用不同的奖励模型，最终在参数梯度上体现为梯度系数(Gradient Coefficient)的不同。作者将奖励函数分为两种类型：1) Rule; 2) Model。Rule方法就是通过硬性的规则根据模型输出的正确性判断输出的质量，Model方法就是训练一个能够对模型输出评分的奖励模型，奖励模型的训练集则来自于规则判断。

作者从实验结果中主要总结了3点结论：

  1. Online RFT中奖励函数是Rule，训练过程会根据Rule丢弃负面的模型输出，因此Gradient Coefficient不会有负值；而GRPO中不会丢弃任何的模型输出，Gradient Coefficient有正有负。从实验结果可知，GRPO性能超越Online RFT，体现了正负Gradient Coefficient的重要性
  2. GRPO+PS采用过程监督方式，因此Gradient Coefficient粒度更细(Fine-grained, Step-aware Gradient Coefficient)；而GRPO+OS采用结果监督方式，因此Gradient Coefficient是单一值。从实验结果可知，GRPO+PS的性能超越GRPO+OS的性能，体现了细粒度Gradient Coefficient的重要性
  3. Iterative RL的方法会增强模型的性能

为什么 RL 有效

DeepSeekMath-RL 的训练和评估章节中提到，只使用SFT数据集的一个子集进行RL训练就可以增强DeepSeekMath-Instruct模型的性能，作者进一步探索了这个结论背后的原因。作者在GSM8K和MATH这两个Benchmark上分别对DeepSeekMath-Instruct-7B和DeepSeekMath-RL-7B进行了评估，评估指标为Pass@K和Maj@K。

RL enhances Maj@K’s performance but not Pass@K.

从实验结果中可以看到，RL强化了模型的Maj@K性能，而不是Pass@K性能。

These findings indicate that RL enhances the model’s overall performance by rendering the output distribution more robust, in other words, it seems that the improvement is attributed to boosting the correct response from TopK rather than the enhancement of fundamental capabilities.

先描述以下Maj@K和Pass@K的含义：

  • Maj@K: 对于每个问题，模型生成K次答案，选取出现频率最高的答案，判断该答案是否正确。
  • Pass@K: 对于每个问题，模型生成K次答案，其中只要有一次是正确答案，则表明正确。

因为LLM模型的生成过程是采样过程，因此同一个问题，多次提问会得到不同的输出，这就是LLM生成多样性的原因。但对于数学问题而言，我们希望模型能够更稳健的输出正确答案，简单来说就是多次提问后模型能够稳定的输出同一个正确答案，而非多样性输出。

从指标的定义上可以看到，Maj@K的提升需要模型能够在K次生成中稳定的尽可能多的生成相同的正确答案；而Pass@K的提升只需要模型在K次生成中提高能生成出正确答案的概率。Pass@K的提升只能保证K次生成中有正确答案的概率增大，而不能保证模型真的能稳定的多次生成相同的正确答案。而Maj@K的提升表明模型除了能够生成正确答案之外，对正确答案的确定性更强，输出分布更集中。

于是，从实验结果中可知，RL增强指令模型性能是通过增强输出分布的鲁棒性来提升模型的整体性能；换句话说，从Pass@K上看，模型生成正确答案的能力并未提升，但从Maj@K上看，模型对正确答案的自信度更高，RL更倾向于让模型在原本回答正确的基础上能够更稳定。

如何提升 RL 的训练效率

作者以RL统一范式的3个要素为基础讨论如何提升RL效率。

Data Source

本文中，作者只使用了SFT数据集中的一部分问题，并通过核采样(Nucleus Sampling, Top-P Sampling)的方式生成问题的答案。未来，作者希望通过采用 OOD问题(Out-of-distribution Question Prompts) 配合更 高阶的采样(解码)技术 生成问题的答案，比如Tree-search Methods[3]。同时采用 高效的推理框架 提升Policy探索的效率，比如论文[4][5][6][7]。

这里解释以下核采样的含义，核采样也叫Top-P Sampling，它是从Top-K Sampling的基础上被提出来的。Top-K Sampling的具体过程如下：

  1. 模型输出词表中每个Token的概率
  2. 按照Token的概率从大到小排列
  3. 取出概率最大的K个Token
  4. 对这K个Token的概率进行Re-scale生成新的概率分布
  5. 从这K个Token中采样Next Token

在Top-K Sampling中即使Next Token的确定性非常大，但每次采样的窗口大小仍然是固定的。

Top-P Sampling的过程如下：

  1. 模型输出词表中每个Token的概率
  2. 按照Token的概率从大到小排列
  3. 取出概率和不超过P的Token
  4. 对这些Token的概率进行Re-scale生成新的概率分布
  5. 从这些Token中采样Next Token

在Top-P Sampling中，采样窗口会根据Next Token的确定性自适应调整，当Next Token确定性很大时，窗口变小，当Next Token确定性很小时，窗口变大，允许模型发挥更大的多样性。

Algorithms

本文中，Gradient Coefficient被直接作用在参数梯度上，这说明本文的所有方法都无条件的完全信任Gradient Coefficient。但实际中并非如此，即奖励函数生成的奖励未读稳定且可靠。所以在未来，作者希望探索能够抵抗噪声奖励信号(Noisy Reward Signals)的RL算法，比如WEAK-TO-STRONG Alignment Methods[8]。

Reward Function

作者提出了3个奖励模型的改进方向：

  1. 如何增强奖励模型的泛化能力，让模型有能力泛化到OOD的问题以及拥有高阶的解码策略
  2. 如何反应奖励模型的不确定度
  3. 如何建立高质量的过程奖励模型

结论

主要记录两点：

  1. DeepSeekMath-RL模型在几何和定理证明任务上弱于闭源模型
  2. 未来作者希望强化数据选择Pipeline用于生成更高质量的预训练数据
