Sarsa与Q-Learning的差异

差异一

虽然Sarsa和Q-Learning都是Value-based的方法，并且都使用TD Learning进行优化，但两者之间存在一些差异。总的来说，Sarsa算法的目的是训练动作价值函数(Action-Value Function) ，而Q-Learning算法的目的是训练最优动作价值函数(Optimal Action-Value Function) 。从两个算法计算TD Target的部分可以明显的看到差异，在Sarsa中，TD Target为，而Q-Learning中TD Target为。两者形式上比较相似，但Sarsa中选择没有通过最大化Q选择，而Q-Learning则使用最大化选择动作。

差异二

Q-learning: try to evaluate value function for greedy policy while following policy, therefore off-policy.

Sarsa: try to evaluate value function for policy while following policy, therefore on-policy.

上述描述来自网络，已记不起来源

实际上强化学习中还有On-Policy和Off-Policy的概念，On-Policy是指产生训练数据(经验)的Policy和实际需要优化的Policy是同一个；Off-Policy是指产生训练数据(经验)的Policy和实际需要优化的Policy不是同一个。

对于Sarsa而言，在计算TD Target 时，是根据Q函数使用得到的，而环境交互时同样使用选择动作，两者是同一策略。因此属于On-Policy范畴。

对于Q-Learning而言，在计算TD Target 时，是根据Q函数使用greedy得到的，而环境交互时却使用选择动作，也就是说Q函数更新时的动作选择策略与环境交互时的动作选择策略不是同一策略。因此属于Off-Policy范畴。

Greedy & Epsilon Greedy

采用Greedy方法选择动作时，选择能够使价值函数Q最大的动作。

采用Epsilon Greedy选择动作时按照公式策略进行，具体含义为的概率通过Q函数最大值选择动作，的概率随机选择动作。

通常，随着训练的进行，的值逐步减小。因为采用的策略其实是对于探索(Exploration)和利用(Exploitation)的一种tradeoff。在训练的一开始，我们希望Agent能够尽可能的探索不同的动作，这样能够有更大的几率学到最优动作。但当训练进行到后期，Agent越来越完美，可以根据以往的经验选择最合适的动作。

Multi-Step TD Target

Multi-Step TD Target是对TD算法的一种改进，这里总结一下Sarsa和Q-Learning算法的目标，如下表所示。

Sarsa Q-Learning
训练 Action-Value Function 训练 Optimal Action-Value Function
TD Target TD Target

表中红色部分的奖励只有时刻这一项，所以标准的TD Target也被称为One-Step TD Target。如果计算TD Target的过程中累积多个奖励，则被称为Multi-Step TD Target。回到TD Learning的核心思想，即公式所示。将公式约等号右侧的部分称为TD Target的原因是包含了真实的奖励项，于是右侧得到的计算结果比左侧更靠谱。从直觉上来看，当右侧累积的奖励更多时，其准确性会更高，因此会提升模型的性能。实验证明，Multi-Step TD Target方法确实性能更优。
