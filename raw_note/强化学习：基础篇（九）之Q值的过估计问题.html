DQN 的高估（Over-estimation）问题

在讲过估计问题之前需要先引入“自举”的概念，以下是TD Learning的过程：

  1. 观察到状态转移四元组
  2. 计算 TD Target
  3. 计算 TD Error
  4. 计算梯度
  5. 梯度下降

在上述的过程中，当第2步计算TD Target时，我们使用DQN估计的值，因此明就是借助DQN估计得到的。同理，第3步中也是通过DQN估计得到的。所以，简单来说，TD Learning方法实际上是使用DQN在时刻的估计去估计DQN在时刻的估计，而这种使用自身估计自身的行为就称为自举（Bootstrapping）。

正因为这种自举的行为，使得使用TD算法训练DQN会产生Over-estimation的问题，造成DQN高估问题的原因可以总结为两个：

  1. 计算TD Target时的最大化运算容易导致会大于真实的动作价值
  2. 因为Bootstrapping的行为，导致高估问题会随着训练过程不断累计

问题二

由于问题一的存在，在DQN的训练过程中形成了正反馈，最终导致这种高估现象不断累积。

为什么高估是缺陷

从TD Learning的方法来看，每次计算TD Target时都选择最大的动作价值，准确的说这里的最大其实是相对的。因此只要高估是均匀的，高估本身其实并没有问题。简单来说就是假如每个动作都同时不断的被高估，那么此时并不会影响Policy动作的选择。但当高估非均匀时，高估就成为了缺陷。而实际中，动作的选择并非是均匀的，这就导致有的动作被多次高估，有的动作被高估的很少，进而导致Policy选择动作出现偏差。比如下表所示的例子：

从产生高估的两个问题出发，解决方案可以有2种方法：

  1. 避免Bootstrapping的发生，使用Target Network计算TD Target[1]
  2. 避免Maximization的使用，使用Double DQN来避免最大化[2]

Target Network

Target Network是一种缓解过估计的方法，它的思想很简单：DQN中使用1个网络来近似最优动作价值，而在Target Network方法中使用2个网络，它们具有不同的作用，其中DQN的用途是收集经验，而Target Network的作用是计算TD Target。

实作中，通常Target Network与DQN的结构相同但参数不同。使用DQN 控制Agent与环境交互并收集经验，使用Target Network 计算TD Target 。

算法流程如下：

  • 观察到Transition
  • 计算TD Target
  • 计算TD Error
  • 梯度下降

简单来说，DQN选择的动作未必与Target Network选择的动作相同。如果全部使用DQN选择动作后估计TD Target，则会导致DQN高估；如果全部使用Target Network选择动作后估计TD Target，则会导致Target Network高估。但如果使用DQN选择动作，放到Target Network中估计 TD Target，可能很大程度上并不是Target Network中的最优动作值，所以缓解了高估效应。
