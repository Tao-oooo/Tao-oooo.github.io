Advantage Actor-Critic (A2C)

在《强化学习：基础篇（十一）之Baseline》笔记中讲到了Baseline的概念，并且将Baseline应用到了REINFORCE算法上。在本笔记中，我们将Baseline应用到Actor-Critic方法上，于是就产生了Advantage Actor-Critic (A2C)方法。

Actor-Critic方法中有两个神经网络，一个称为Actor Network (Policy Network)，另一个称为Critic Network (Value Network)。这里需要注意的是，A2C方法与Actor-Critic方法有一些差异，后者的Critic Network是对动作价值函数做近似，而前者是对状态价值函数做近似。同时，由于状态价值函数只依赖于状态，所以相比于对动作价值函数做近似而言更容易训练。

其中，表示当前状态的胜算，如果越好，则胜算越大，即V值越大。同样地，对于而言，如果越好，则胜算越大，即V值也越大。而与动作是无关的，因为看到这个状态时还没有选择做哪个动作。而则与动作有关，因为选择动作之后才会产生状态。所以如果选择的动作越好，那么状态也就越好，那么这一项会是正的；反之，如果选择的动作不好，那么状态就会变差，此时这一项就会是负的。所以两者之差会反应动作带来的优势。

REINFORCE with Baseline 与 A2C

详细比较两个算法的训练过程。

REINFORCE with Baseline的训练过程：

  • 观测到一个完整的轨迹
  • 计算回报，并且计算
  • 更新Policy Network网络参数
  • 更新Value Network网络参数
  • 从重复第 2、3、4 步

A2C 的训练过程：

  • 观测到个Transition
  • -Step TD Target计算
  • TD Error计算
  • 更新Policy Network参数更新
  • 更新Value Network参数更新

两者的差异主要在标红部分，其中REINFORCE with Baseline中的是真实观测到的折扣回报数据，而A2C中的则包含了两部分，其中一部分使用了真实观测到奖励数据。

REINFORCE with Baseline

从上表可以看到，REINFORCE with Baseline看起来是A2C算法的特例。它们的网络结构一致，但是Value Network的作用不一样。REINFORCE with Baseline中的Value Network只是一个Baseline，用来降低Policy Gradient的方差，加快收敛速度。而A2C中的Value Network称为Critic，作用是评价Action的好坏。

同时，由于REINFORCE with Baseline使用的全是真实观测数据，所以没有使用Bootstrapping，而A2C中由于引入了Value Function进行估计，因此使用了Bootstrapping。
